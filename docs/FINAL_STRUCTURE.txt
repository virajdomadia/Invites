================================================================================
                          KEY CONCEPTS
================================================================================

ROUTE GROUPS:
  • Folder names with parentheses: (auth), (home)
  • Don't affect URLs: (auth)/login → /login (NOT /(auth)/login)
  • Organize screens without changing routes
  • Each group has its own layout

FEATURES:
  • Contains all logic for a feature
  • Organized by feature, not by type
  • Each feature has: components, screens, services, hooks
  • Features export clean APIs

CORE:
  • Shared utilities used by all features
  • API client, hooks, constants
  • Features depend on core
  • Core never depends on features

SEPARATION:
  • Routing: app/ folder handles navigation
  • Logic: features/ folder handles business logic
  • Utilities: core/ folder handles shared stuff

================================================================================
                        BEST PRACTICES
================================================================================

✅ DO:
  • Use route groups to organize routes
  • Keep routing logic in app/ folder
  • Keep feature logic in features/ folder
  • Keep shared utilities in core/ folder
  • Use simple, descriptive route names
  • Export features with clean APIs
  • Use relative imports within features
  • Keep features independent

❌ DON'T:
  • Use parentheses in router.push() calls
  • Put feature logic in app/ folder
  • Put routing logic in features/ folder
  • Create circular dependencies
  • Use uppercase in group names
  • Mix routing with business logic
  • Import deeply - use exports
  • Create features that depend on other features

================================================================================
                        EXAMPLE: ADD NEW FEATURE
================================================================================

Let's add a "Profile" feature:

1. Create folder:
   mkdir -p src/features/profile/{components,screens,services,hooks}

2. Create services:
   src/features/profile/services/profile-service.ts

3. Create components:
   src/features/profile/components/profile-card.tsx

4. Create screens:
   src/features/profile/screens/index.tsx
   src/features/profile/screens/edit.tsx

5. Create hooks:
   src/features/profile/hooks/useProfile.ts

6. Create exports:
   src/features/profile/index.ts

7. Create routes:
   mkdir -p src/app/\(profile\)
   src/app/(profile)/_layout.tsx
   src/app/(profile)/index.tsx
   src/app/(profile)/edit.tsx

8. Use in app:
   import { useProfile, ProfileCard } from '../../features/profile';

Now you have:
  Routes:     /profile, /profile/edit
  Features:   profile/ with all logic
  Ready:      to add more screens/features
